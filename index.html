About.

  • 28 years old
  • From the Netherlands
  • Graduated in 'Art: Decoration and Restoration'
  • Worked with GML, C#, HTML, CSS, Javascript, Angular, React, p5.js library, JSON

As a kid, I could be found either with a pencil in my hand or sitting in front of a PC, but my studies have always been oriented towards more artistic endeavors. My interest in programming, triggered by an HTML-course I did during my study, came as a pleasant, yet fitting, surprise!

After I graduated from the study 'Art: Restoration and Decoration' I was left unsatisfied. It started to dawn on me that art might not be the best career choice. Obviously, because it doesn't offer a stable income, but mainly because I started to realize I prefer to keep it as a hobby, allowing me to have full control and create whatever I want.

When I started programming, I discovered it doesn't just allow me to be creative but it also scratches an entirely different part of my brain. It allows me to think analytically and create structure and challenges me to solve complex problems. With every new little thing I learn my enthusiasm and motivation is re-energized which often sparks several new project ideas.

Art, for me, is more a means of expression and creative outlet. Programming is a skill I want master.

As of now I have mostly been focusing on improving my Front-End skills, but with the start of my study I'll also be diving into the Back-End.

Projects.

Since a study used to be no option due to financial reasons, two years ago I started teaching myself programming with the help of the internet. However, before I decided to pursue it more seriously and shifted my focus towards Front-End Development, I made some projects in GameMakerLanguage and have worked a little in C#.

Throughout these two years, I've made multiple personal projects, including this website which I'm currently planning to re-design. Below you can find links to some of my projects along with a short description. Most of these projects were made with the goal in mind of learning a specific language or framework. Some are simply passion projects or experiments.

GameMaker.

My first projects were written in GML, using GameMaker Studio 2. I've made several small projects consisting of modular game mechanics that can be used by other creators in their own game-related projects.

Even though I stopped working in GameMaker Studio quite some time ago, my projects are still being downloaded today, currently estimating at 1300+ downloads.

Among other things, working with GML gave me a basic understanding of if-statements, for-loops, switches, and finite-state machines. It also taught me the importance of project scope and the dangers of feature creep.

Go to my itch.io.

MapGen Bronze (C#).

One of the bigger projects I have made in C# as Console Application is a 2D Map Generator using Simplex Noise to generate it's landmass and biomes. The generated maps can be saved (and loaded) as JSON files and can be navigated using the arrow-keys, which allows for the selection of individual grid-cells and reading out their data.

This project taught me my first lessons in C#, the importance of keeping my code organized and made me familiar with the use of libraries.

Go to Project Page.

Generative Art (p5.js).

My interest in generative art led me to try out the p5.js library. After a few experiments, I made a simple generator that uses the Random-Walker technique to generate and draw visuals (influenced by several adjustable variables) onto a HTML Canvas.

Working on this project taught more about organizing, refactoring and commenting my code.

Go to Live Project.

Dice Roller (Angular).

To learn more about Javascript frameworks I made a project in Angular 8, namely, a digital Dice Roller with a variety of die to choose from.

Making this Dice Roller showed me the value of Javascript frameworks, made me more familiar with HTML and CSS and was my first attempt at responsive design.

Go to Live Project.

MapGen Gold (p5.js).

When I began to become more comfortable with HTML, CSS, and JS, I started wondering if I could recreate my Map Generator using the p5.js library. This time using sprites for visuals instead of simple symbols and letters. This went easier than expected and the project turned out far better than my C# Map Generator.

It uses Perlin Noise to generate its maps, has a menu to manipulate the generation process, and has different visual options. There's still a lot to improve but it has been a great learning experience.

The size of this project made it a challenge to keep things organized and while I still struggle with this skillset, I'm looking forward to learning more about it. Working on this, I gained more knowledge about HTML, CSS, Javascript, and p5.js. I also learned more about binding UI to logic and dynamically creating HTML input elements.

Go to Live Project.

Template Canvas UI (HTML/JS).

Since I started making more HTML Canvas based projects, I decided to create a Responsive Template UI that makes it easier to hook up a menu to a canvas project's logic.

Looking back now it might have been better to create this program in a framework like React, but working on it has definitely taught me something. It deepened my understanding of responsive design and taught me more about modular design.

Go to Live Project.

Minesweeper Clone (React JS).

Having struggled a little with Angular and knowing that React was another popular JS framework made me curious to learn more about it. To get more familiar with communication between components, a friend advised me to try making a Minesweeper clone with several difficulty options.

This project made me more capable of using React and taught me how to better organize my code into smaller functions. I definitely favor React over Angular now!

Go to Live Project.

Github Hosted Blog (HTML, CSS, JS, JSON).

A while ago I was rethinking my website and considered adding a blog. I chose not to use any existing tools like WordPress, since I wanted to make it myself, for fun and for practice.

Even though I haven't added the blog to my site yet, due to having no time to write any content, it's functional and doesn't need a server to store its posts, other than it's own Github repository.

This project taught me how to work with constraints (having limited time to learn about servers and storing data) and come up with creative solutions.

Go to Live Project.
